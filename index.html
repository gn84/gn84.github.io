Wasserstein Distance: a Geometric metric for AI?

Giuseppe Napoli, 25 June 2026.

Almost every metric we use in machine learning compares numbers. Accuracy is a count. A loss is an average. Even in algorithmic fairness, we usually look at scalar differences. Optimal transport asks a fundamentally deeper question: given two entire probability distributions, what is the mathematically cheapest way to morph one into the other?

This same lens is the right one in a surprising number of domains, from straightening the trajectories of state-of-the-art generative models to rigorously auditing algorithms for bias.

From Monge to Kantorovich

In 1781, Gaspard Monge asked a highly literal question: if you have a pile of sand shaped like A and you need to reshape it into B, what is the most efficient plan? You must move every grain, and moving a grain costs in proportion to the distance it travels. The minimum total cost of that reshaping is a distance between the two shapes, historically known as the Earth Mover's Distance.

Monge wanted a direct map sending each source point to a single target point. The math here is brutal: such a map might not exist (you can't split a single atom of mass with a pure function), and the optimization is strictly non-convex.

In 1942, Leonid Kantorovich relaxed the problem beautifully. Instead of a strict map, he proposed looking for a transport plan γ, a joint distribution that allows the mass of one point to be split across multiple destinations. The only constraints are the marginals.

Wp(μ, ν) = ( infγ ∈ Π(μ,ν) ∫ c(x,y)p dγ(x,y) )1/p

With a Euclidean cost and p = 2, you get the 2-Wasserstein distance (W₂). This is a true metric on the space of probability distributions, symmetric and obeying the triangle inequality. Unlike a pseudo-divergence, its unit matches the unit of the domain itself, making the distance physically and mathematically interpretable.

Why KL Divergence Fails

Take two narrow, identical bumps, one on the left and one on the right, with no overlap. Kullback-Leibler (KL) divergence and Total Variation see them as maximally different, regardless of how far apart the peaks actually are. They compare densities vertically, point by point, and instantly saturate where one density is zero and the other isn't.

Wasserstein, instead, measures how far you must slide the mass horizontally to line them up. It grows smoothly as the peaks drift apart.

This horizontal perspective is exactly why optimal transport resurrected deep generative models via the Wasserstein GAN (Arjovsky et al., 2017). By swapping Jensen-Shannon for W₁, the generator receives a clean, usable gradient even when real and fake samples live on completely disjoint manifolds, the exact regime where classic GANs collapse. The dual form makes this computationally practical:

W1(μ, ν) = sup‖f‖L ≤ 1 ( Eμ[f] − Eν[f] )

The Computational Bottleneck and the Sinkhorn Fix

The discrete Kantorovich problem is a linear program with O(n³ log n) complexity. On a standard 50k deep learning batch, it is dead on arrival.

Marco Cuturi (2013) changed the game entirely with entropic regularization:

OTε(μ, ν) = infγ ( ∑i,j γij Cij − ε H(γ) )

Adding an entropy term makes the objective strictly convex. The solution factorizes, reducing the entire optimization to Sinkhorn iterations: simple, alternating matrix-vector products that normalize rows and columns. This is massively parallelizable on GPUs, end-to-end differentiable, and fits into a few lines of JAX:

import jax, jax.numpy as jnp
from jax import jit, lax
from jax.scipy.special import logsumexp

@jit
def sinkhorn(C, a, b, eps=0.01, n_iter=200):
    # C: (n,m) cost matrix; a, b: marginals
    log_a, log_b = jnp.log(a), jnp.log(b)
    f, g = jnp.zeros_like(a), jnp.zeros_like(b)

    def step(carry, _):
        f, g = carry
        f = eps * (log_a - logsumexp((g[None, :] - C) / eps, axis=1))
        g = eps * (log_b - logsumexp((f[:, None] - C) / eps, axis=0))
        return (f, g), None

    (f, g), _ = lax.scan(step, (f, g), None, length=n_iter)
    P = jnp.exp((f[:, None] + g[None, :] - C) / eps)
    return jnp.sum(P * C), P

The 1D Exploit

For one-dimensional distributions (like scalar model predictions, age distributions, or any single continuous feature) optimal transport offers a massive computational shortcut. In 1D, the optimal map simply matches the quantiles. You sort both arrays and pair them up.

W2(μ, ν)2 = ∫01 ( Qμ(t) − Qν(t) )2 dt

There is no ε, no entropic bias, and no hyperparameters. Just a sort. In JAX, it is exact, vectorized, and practically free:

@jit
def w2_1d(x, y, n_q=512):
    # exact 1D W2: pair the quantiles, no regularization
    q  = (jnp.arange(n_q) + 0.5) / n_q
    qx = jnp.quantile(x, q)
    qy = jnp.quantile(y, q)
    return jnp.sqrt(jnp.mean((qx - qy) ** 2))

When you need to scale back up to multivariate cases (feature spaces, barycenters), Cuturi's JAX-native OTT library handles the geometries and provides the debiased Sinkhorn divergence in three lines:

from ott.geometry import pointcloud
from ott.tools import sinkhorn_divergence

# S_eps(a,b) = OT_eps(a,b) - 0.5*OT_eps(a,a) - 0.5*OT_eps(b,b)
out = sinkhorn_divergence.sinkdiv(pointcloud.PointCloud, Xa, Xb, epsilon=0.01)
S_eps = out.divergence

Generative AI and Flow Matching

Once you have a geometric, differentiable distance between distributions, the applications multiply. The most powerful modern use case is Flow Matching (Lipman et al., 2022).

Instead of learning a score function like standard diffusion models, you learn a velocity field that transports noise into data. By coupling samples via minibatch OT during training, you physically straighten the trajectories of the flow. This allows the model to generate high-fidelity samples in radically fewer steps, pushing the bleeding edge toward one-step generation (Optimal Flow Matching, 2024).

Fairness: Diagnostics over Mass Destruction

Optimal transport has also revolutionized algorithmic fairness, mostly in a corrective form (e.g., "total repair" methods that move a disadvantaged group's distribution toward a Wasserstein barycenter to force demographic parity).

While mathematically elegant, blindly repairing data by moving mass is dangerous. Pushing samples toward a barycenter can force them off the true data manifold and into extrapolation regions the model was never trained on.

A more rigorous approach uses W₂ strictly as a diagnostic certificate. Instead of mutating the underlying data, the Wasserstein distance measures exactly how far a protected group's distribution sits from a fair reference distribution. Because W₂ retains the unit of the domain, it yields a single, comparable figure that can track bias across different models, datasets, and iterations, bypassing the fragility of scalar fairness metrics (like disparate impact ratios) that break down when base rates shift.

The Horizon

The framework has real open problems. The curse of dimensionality is severe: estimating W₂ from high-dimensional samples scales poorly (n−1/d), requiring massive amounts of data. Entropic bias must be carefully managed using the Sinkhorn divergence, and unbalanced OT is still required to properly handle datasets with extremely small minority populations.

Still, the core point stands: while standard metrics offer weak, incommensurable proxies for structural differences, optimal transport provides a true, unit-aware geometric distance. Whether you are straightening generative flows or certifying algorithmic fairness, W₂ is the ultimate measuring stick.
